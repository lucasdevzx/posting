Posts - Recursos

GET

/posts?page=0&size=0

Responsável por obter todos os posts através de paginação.

/posts/me?page=0&size=0

Responsável por obter todos os posts do usuário atual atráves de paginação.

POST

/posts

Responsável por criar um post para o usuário atual.

DELETE

/posts/{id}

Responsável por deleter um post através do ID.

PUT

/posts/{id}

Responsável por atualizar um post atráves do ID.

Requisição JSON

Posting exige um padrão para o envio de informações para processamento.
O meio utilizado é o formato JSON. Para enviar dados para Posting é necessário seguir o modelo abaixo:

JSON

    {
    "name": "",
    "description": ""
    }

Resposta JSON

Posting envia como resposta um padrão com as informações processadas.
O meio utilizado é o formato JSON. O modelo de resposta é o informado abaixo:

JSON

    {
    "id": 0,
    "author": {
        "id": 0,
        "name": ""
    },
    "title": "",
    "description": "",
    "category": "",
    "date": "",
    "permissions": {
        "canEdit": true,
        "canDelete": true,
        "isOwner": true
      }
    }
